# TEMPORA Theory Assumptions

## General Position

TEMPORA treats mathematical statements as scoped claims about explicit model
classes and numerical procedures. It does not infer broad semantic guarantees
from empirical behavior.

## Dynamical-System Assumptions

- Input trajectories are generated by controlled synthetic systems.
- Time grids are finite and reproducible.
- Numerical integration tolerances are part of the experimental protocol.
- Arrays must be finite; NaN and Inf values are failures.

## Model Assumptions

- The latent dynamics are evaluated for the MVP model family:

```text
dz/dt = -D z + W tanh(z) + B phi(u(t)) + b
```

- The damping vector `D` is strictly positive.
- The activation has known Lipschitz constant `L_sigma`; for `tanh`,
  `L_sigma <= 1`.
- Recurrent weights are constrained through spectral-norm projection.
- Claims about contraction are based on sufficient, not necessary, conditions.

## Contraction Claim Boundary

The MVP may claim contraction only under:

```text
min(D) > L_sigma * ||W||_2 + margin
```

and only for the latent dynamics with respect to latent state `z` under the
documented assumptions.

## Plasticity Assumptions

- Local plasticity updates are followed by projection into the contractive set.
- Stability claims concern the post-projection parameter state.
- Learning dynamics are not claimed to be globally optimal.

## Topological Metrics Assumptions

- Persistent homology is computed on finite point clouds.
- Bottleneck or Wasserstein distances are empirical summary metrics.
- Similar persistence diagrams do not prove semantic equivalence.
- The metrics are used to compare controlled input and latent trajectories, not
  arbitrary real-world data.

## Documentation Rule

Every future theorem document must include statement, assumptions, proof sketch,
implementation correspondence, empirical checks, limitations, and related tests.
